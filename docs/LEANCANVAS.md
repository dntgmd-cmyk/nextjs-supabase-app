# Lean Canvas — 모임 이벤트 관리 웹 MVP

---

## 1. 문제 (Problem)

> 해결하려는 상위 3가지 문제

1. **분산된 도구** — 공지는 카카오톡, 정산은 더치트, 참여자 관리는 엑셀/메모로 파편화되어 주최자 업무 부담이 큼
2. **정산 분쟁** — 수기 계산 오류, 납부 여부 추적 불가로 모임 후 금전 갈등 발생
3. **카풀 조율 비효율** — 단체 채팅방에서 카풀 매칭 시 메시지가 묻혀 운전자·탑승자 연결 실패

**기존 대안**

- 카카오톡 단체채팅 (공지·카풀 혼재, 기록 유실)
- 네이버 폼 (단방향, 승인/거절 불가)
- 더치트 / 정산나우 (정산 전용, 모임 통합 불가)

---

## 2. 고객 세그먼트 (Customer Segments)

> 타겟 고객

**주요 고객: 모임 주최자**

- 수영·헬스·러닝 등 정기 운동 모임 운영자
- 동호회·소모임 총무 역할을 맡은 사람
- 2~30명 규모 모임을 월 1회 이상 정기 운영하는 사람

**부가 고객: 모임 참여자**

- 주최자가 초대한 모임 멤버
- 참여 신청·카풀 탑승·정산 납부를 처리해야 하는 사람

**얼리 어답터**

- 기존에 스프레드시트·메모앱으로 모임을 관리하던 주최자
- 정산 분쟁을 경험해본 소모임 총무

---

## 3. 고유 가치 제안 (Unique Value Proposition)

> 한 문장으로 전달하는 핵심 가치

**"모임 주최자의 공지·참여자·카풀·정산, 한 곳에서 끝내세요"**

- 모임 생성부터 정산 마감까지 하나의 서비스로 완결
- 주최자는 승인/공지/정산 클릭 한 번으로 처리, 참여자는 이메일로 즉시 안내
- 도구를 바꿀 필요 없이 링크 하나로 멤버 전원 접근 가능

---

## 4. 솔루션 (Solution)

> 문제별 핵심 해결책 (상위 3가지)

| 문제        | 솔루션                                                     |
| ----------- | ---------------------------------------------------------- |
| 도구 분산   | 공지·참여자·카풀·정산을 탭 하나에서 통합 관리              |
| 정산 분쟁   | 비용 항목 입력 → 1/N 자동 계산 → 납부 체크 → 이메일 요청   |
| 카풀 비효율 | 운전자 등록(좌석 수) → 탑승 신청 → 확정/거절 → 이메일 알림 |

---

## 5. 채널 (Channels)

> 고객에게 도달하는 경로

**획득 채널**

- 주최자가 참여자에게 모임 링크 공유 (바이럴 핵심)
- 수영·헬스 커뮤니티(네이버 카페, 오픈카톡방) 입소문
- 소모임 총무 커뮤니티 타겟 콘텐츠 마케팅

**활성화 채널**

- 이메일 알림 (참여 승인, 공지, 정산 요청) — 재방문 유도
- 모임 링크 공유로 자연 유입 (참여자 → 신규 주최자 전환)

---

## 6. 수익원 (Revenue Streams)

> MVP 이후 수익 모델 (현재는 무료)

| 단계 | 모델          | 내용                                                     |
| ---- | ------------- | -------------------------------------------------------- |
| MVP  | 무료          | 기능 검증 및 사용자 확보                                 |
| v2   | 프리미엄 구독 | 월 N천원 — 모임 수 무제한, 이메일 발송 무제한, 파일 첨부 |
| v3   | 팀 플랜       | 동호회·클럽 단위 구독, 관리자 다수 지정                  |

**MVP 단계 검증 지표**: 주최자 재사용률, 참여자 1인당 초대 수

## 8. 핵심 지표 (Key Metrics)

> 제품 건강성을 측정하는 핵심 수치

| 지표                   | 정의                             | 목표 (MVP 3개월) |
| ---------------------- | -------------------------------- | ---------------- |
| 주최자 활성 수         | 월 1회 이상 모임 생성한 사용자   | 50명             |
| 모임당 평균 참여자 수  | 승인된 참여자 수 / 모임 수       | 8명 이상         |
| 정산 완료율            | 정산 생성 후 전원 납부 완료 비율 | 70% 이상         |
| 주최자 재방문율        | 2회 이상 모임 생성한 주최자 비율 | 50% 이상         |
| 참여자 → 주최자 전환율 | 참여자로 시작해 모임을 만든 비율 | 10% 이상         |

---

## 9. 경쟁 우위 (Unfair Advantage)

> 쉽게 복사하거나 구매할 수 없는 강점

1. **모임 특화 워크플로우** — 범용 협업 툴이 아닌 소/중규모 정기 모임에 최적화된 흐름 (신청→승인→공지→카풀→정산)
2. **링크 기반 바이럴** — 주최자가 참여자를 초대할 때 자연스럽게 서비스 확산, 참여자가 새 주최자로 전환
3. **낮은 진입 장벽** — 앱 설치 없이 이메일/구글 로그인만으로 즉시 사용, 참여자에게 별도 가입 부담 최소화
